## Why

The project needs a concrete MVP that delivers immediate user value and establishes the baseline behavior for a task management product. A minimal task workflow should be specified now so implementation can proceed with a clear product scope and acceptance criteria.

## What Changes

- Add an MVP task management capability that lets users create tasks, view a task list, mark tasks as completed, and delete tasks.
- Define the expected task data and user-visible behavior for each core action.
- Establish a simple, single-user workflow with no advanced collaboration, filtering, or scheduling features in this change.

## Capabilities

### New Capabilities
- `task-management`: Core task lifecycle for creating, listing, completing, and deleting tasks in the MVP.

### Modified Capabilities
- None.

## Impact

- Adds a new product capability spec under `openspec/specs/task-management/`.
- Establishes the scope for the first implementation tasks and design decisions for the task management app.
- No external APIs or integrations are required for this MVP proposal.
